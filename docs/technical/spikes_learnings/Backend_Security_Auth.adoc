## Zu beantwortende Frage

**Leitfrage:** Wie übergeben wir die Nutzeridentität sicher vom React-Frontend an das Django-Backend, damit eine eingehende Ausleihanfrage eindeutig "Nina" als Ausleiherin zugeordnet werden kann?

---

## Hintergrund & Motivation

Nina möchte ihre *eigenen* Ausleihanfragen einsehen können. Wenn sie eine Anfrage an das Backend sendet, muss Django wissen, wer dieser Nutzer ist (Identifikation). Da React und Django getrennt sind, müssen wir klären, wie wir Session-Tokens oder JWTs generieren und beim POST-Request sicher mitsenden.

**Blockierte Story / Entscheidung:**
Blockiert die User Stories:
- "Als Nina Nutzerin möchte ich eine Ausleihanfrage... erstellen können"
- "... den aktuellen Bearbeitungsstatus meiner Ausleihanfragen einsehen..."
